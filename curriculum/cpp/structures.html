C++ Structures

Structures in C++ are user-defined data types that allow you to group related data items of different types together. They are fundamental to organizing complex data and are the foundation for object-oriented programming in C++.

What is a Structure?

A structure is a collection of variables (called members) that can be of different data types, grouped together under a single name. Structures help organize related data and make code more manageable.

Why Use Structures?

  • Organization: Group related data together
  • Readability: Make code more self-documenting
  • Reusability: Define once, use multiple times
  • Modularity: Encapsulate related functionality
  • Type Safety: Prevent mixing incompatible data

Defining a Structure

Use the struct keyword to define a structure:

// Basic structure definition
struct Person {
    std::string name;
    int age;
    double height;
    char gender;
};

// Structure with different data types
struct Rectangle {
    double length;
    double width;
};

// Structure with arrays
struct Student {
    std::string name;
    int grades[5];  // Array of 5 integers
    double gpa;
};

Declaring Structure Variables

After defining a structure, you can create variables of that type:

// Method 1: Declare after definition
struct Person {
    std::string name;
    int age;
};

Person person1;  // Create a Person variable

// Method 2: Declare at definition time
struct Car {
    std::string make;
    std::string model;
    int year;
} myCar, yourCar;  // Declare variables here

// Method 3: Using typedef (optional)
typedef struct {
    std::string title;
    std::string author;
    int pages;
} Book;

Book myBook;  // No need for 'struct' keyword

Initializing Structures

// Method 1: Initialize member by member
Person person1;
person1.name = "John Doe";
person1.age = 30;

// Method 2: Initialize with curly braces (C++11)
Person person2 = {"Jane Smith", 25};

// Method 3: Designated initializers (C++20)
Person person3 = {
    .name = "Bob Johnson",
    .age = 35
};

// Method 4: Constructor-like initialization
Person person4{"Alice Brown", 28};

Accessing Structure Members

Use the dot operator (.) to access structure members:

Person person;
person.name = "John";
person.age = 30;

std::cout << "Name: " << person.name << std::endl;
std::cout << "Age: " << person.age << std::endl;

// Calculate something using members
int yearsUntilRetirement = 65 - person.age;

Structures with Functions

You can pass structures to functions and return them:

struct Rectangle {
    double length;
    double width;
};

// Function that takes a structure as parameter
double calculateArea(Rectangle rect) {
    return rect.length * rect.width;
}

// Function that returns a structure
Rectangle createRectangle(double l, double w) {
    Rectangle rect;
    rect.length = l;
    rect.width = w;
    return rect;
}

// Usage
Rectangle myRect = createRectangle(5.0, 3.0);
double area = calculateArea(myRect);
std::cout << "Area: " << area << std::endl;

Nested Structures

Structures can contain other structures as members:

struct Address {
    std::string street;
    std::string city;
    std::string state;
    std::string zipCode;
};

struct Employee {
    std::string name;
    int id;
    Address homeAddress;
    double salary;
};

// Usage
Employee emp;
emp.name = "John Doe";
emp.id = 12345;
emp.homeAddress.street = "123 Main St";
emp.homeAddress.city = "Anytown";
emp.salary = 50000.0;

// Accessing nested members
std::cout << emp.homeAddress.city << std::endl;

Arrays of Structures

struct Student {
    std::string name;
    int id;
    double gpa;
};

// Array of structures
Student classRoster[30];

// Initialize array elements
classRoster[0] = {"Alice", 1001, 3.8};
classRoster[1] = {"Bob", 1002, 3.2};

// Loop through array
for (int i = 0; i < 2; i++) {
    std::cout << classRoster[i].name << ": " << classRoster[i].gpa << std::endl;
}

Structures vs Classes

In C++, structures and classes are very similar:

Aspect Structures Classes
Default access public private
Keyword struct class
Inheritance Supported Supported
Member functions Allowed Allowed

Converting struct to class

// This structure
struct Point {
    int x;
    int y;
};

// Is equivalent to this class
class Point {
public:
    int x;
    int y;
};

Structure Padding and Alignment

C++ may add padding between structure members for memory alignment:

struct Example {
    char c;      // 1 byte
    // 3 bytes padding
    int i;       // 4 bytes
    char d;      // 1 byte
    // 3 bytes padding
}; // Total: 12 bytes (not 6)

Controlling Alignment

// Pack structure to minimize size
#pragma pack(1)
struct PackedExample {
    char c;
    int i;
    char d;
}; // Total: 6 bytes
#pragma pack()  // Reset to default

Example Program

#include <iostream>
#include <string>
#include <vector>

struct Book {
    std::string title;
    std::string author;
    int year;
    double price;
};

struct Library {
    std::string name;
    std::vector<Book> books;
};

void addBook(Library& lib, Book book) {
    lib.books.push_back(book);
}

void displayBook(const Book& book) {
    std::cout << "Title: " << book.title << std::endl;
    std::cout << "Author: " << book.author << std::endl;
    std::cout << "Year: " << book.year << std::endl;
    std::cout << "Price: $" << book.price << std::endl;
    std::cout << "------------------------" << std::endl;
}

int main() {
    Library myLibrary;
    myLibrary.name = "City Library";
    
    // Add some books
    addBook(myLibrary, {"1984", "George Orwell", 1949, 12.99});
    addBook(myLibrary, {"To Kill a Mockingbird", "Harper Lee", 1960, 14.95});
    addBook(myLibrary, {"The Great Gatsby", "F. Scott Fitzgerald", 1925, 10.50});
    
    std::cout << "Books in " << myLibrary.name << ":" << std::endl;
    std::cout << "========================" << std::endl;
    
    for (const auto& book : myLibrary.books) {
        displayBook(book);
    }
    
    return 0;
}

Common Mistakes

  • Forgetting the struct keyword in C (not needed in C++)
  • Not initializing structure members
  • Using wrong member access operator
  • Memory alignment issues
  • Passing large structures by value instead of reference

Best Practices

  • Use meaningful names for structures and members
  • Initialize structures when declaring them
  • Pass structures by reference to functions to avoid copying
  • Use const references for read-only parameters
  • Consider memory layout when performance is critical
  • Document the purpose of each structure
