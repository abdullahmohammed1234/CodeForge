C++ Getting Started

Welcome to your C++ programming journey! This guide will help you set up your development environment and write your first C++ program. C++ is a compiled language, so you'll need to install a compiler and learn the compilation process.

Prerequisites

Before starting with C++, you should have:

  • A computer running Windows, macOS, or Linux
  • Basic understanding of programming concepts
  • Familiarity with using a command line/terminal
  • A text editor or Integrated Development Environment (IDE)

Step 1: Install a C++ Compiler

C++ code needs to be compiled before it can run. Here are the most common compilers:

Windows

  • Visual Studio Community: Free IDE with built-in compiler
  • MinGW: GCC compiler for Windows
  • MSYS2: Unix-like environment with GCC

macOS

  • Xcode: Apple's IDE (install Command Line Tools)
  • Homebrew + GCC: Alternative compiler installation

Linux

  • GCC: Usually pre-installed or available via package manager
  • Clang: Alternative compiler from LLVM project

Step 2: Choose an IDE or Text Editor

While you can write C++ code in any text editor, an IDE provides helpful features:

  • Visual Studio Code: Free, lightweight, with C++ extensions
  • Visual Studio: Full-featured IDE for Windows
  • CLion: Cross-platform IDE from JetBrains
  • Code::Blocks: Free, open-source IDE
  • Eclipse CDT: Eclipse with C++ support

Step 3: Verify Your Installation

After installation, verify your compiler works:

// Windows (Command Prompt)
g++ --version

// macOS/Linux (Terminal)
g++ --version

You should see version information if the compiler is installed correctly.

Your First C++ Program

Let's create the traditional "Hello, World!" program:

#include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}

Understanding the Code

  • #include <iostream>: Includes the input/output stream library
  • int main(): The main function where program execution begins
  • std::cout: Standard output stream (console output)
  • <<: Output operator
  • std::endl: End line (adds newline and flushes output)
  • return 0: Indicates successful program execution

Compiling and Running Your Program

Using Command Line

// 1. Create a file called hello.cpp
// 2. Copy the code above into the file
// 3. Compile the program
g++ hello.cpp -o hello

// 4. Run the executable
./hello  // On Linux/macOS
hello.exe  // On Windows

Using an IDE

Most IDEs have built-in compilation and run commands:

  • Create a new C++ project or file
  • Paste the code
  • Use the "Build" or "Compile" menu option
  • Use "Run" or "Execute" to run the program

Common Compilation Errors

  • Missing semicolon: Every statement must end with ;
  • Missing header: Include necessary headers like <iostream>
  • Case sensitivity: C++ is case-sensitive
  • Missing main function: Every program needs a main() function
  • Wrong file extension: Use .cpp for C++ source files

C++ File Structure

A typical C++ program consists of:

  • Header files: Declarations and function prototypes
  • Source files: Implementation of functions
  • Main function: Entry point of the program
  • Libraries: Pre-compiled code for common functionality

Basic Program Template

// Include necessary headers
#include <iostream>
#include <string>  // For string operations

// Using namespace to avoid std:: prefix
using namespace std;

// Main function
int main() {
    // Your code here
    cout << "Welcome to C++!" << endl;
    
    // Variable declaration and usage
    string name;
    cout << "Enter your name: ";
    getline(cin, name);
    
    cout << "Hello, " << name << "!" << endl;
    
    return 0;
}

Mini Project

Create a simple interactive C++ program:

  • Include necessary headers
  • Use cout for output and cin for input
  • Declare variables to store user input
  • Compile and run the program
  • Bonus: Add basic calculations or string manipulation

Next Steps

Now that you have your environment set up:

  • Learn about variables and data types
  • Study control structures (if-else, loops)
  • Understand functions and modular programming
  • Explore the Standard Template Library (STL)
  • Practice with small projects

Troubleshooting

  • Compiler not found: Check your PATH environment variable
  • Permission denied: Make sure you have write permissions
  • Linker errors: Check for missing libraries or incorrect linking
  • Runtime errors: Use debugging tools to identify issues
