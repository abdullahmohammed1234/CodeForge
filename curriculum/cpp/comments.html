C++ Comments

Comments in C++ are used to add explanatory notes or disable parts of code without affecting the program's execution. They help make code more readable and maintainable for developers.

Why Use Comments?

  • Documentation: Explain what the code does
  • Readability: Make complex logic easier to understand
  • Debugging: Temporarily disable code sections
  • Collaboration: Help other developers understand your code
  • Maintenance: Remind yourself of code purpose later

Types of Comments in C++

C++ supports two types of comments:

  • Single-line comments: Start with // and continue to the end of the line
  • Multi-line comments: Start with /* and end with */, can span multiple lines

Single-Line Comments

Single-line comments begin with two forward slashes (//) and extend to the end of the current line. They are commonly used for brief explanations or to comment out single lines of code.

#include <iostream>

int main() {
    // This is a single-line comment
    std::cout << "Hello, World!" << std::endl; // Comment after code
    
    // You can also use comments to explain variables
    int age = 25; // Age in years
    
    return 0;
}

Multi-Line Comments

Multi-line comments start with /* and end with */. They can span multiple lines and are useful for longer explanations or commenting out blocks of code.

#include <iostream>

/*
 * This is a multi-line comment
 * that spans several lines.
 * It's useful for detailed explanations.
 */

int main() {
    /* 
     * Calculate the area of a rectangle
     * Formula: area = length * width
     */
    int length = 10;
    int width = 5;
    int area = length * width;
    
    std::cout << "Area: " << area << std::endl;
    
    return 0;
}

Best Practices for Comments

  • Be Clear and Concise: Write comments that add value
  • Keep Updated: Update comments when code changes
  • Avoid Obvious Comments: Don't comment self-explanatory code
  • Use Consistent Style: Follow a consistent commenting style
  • Comment Complex Logic: Explain non-obvious algorithms or business rules

Commenting Out Code

Comments can be used to temporarily disable code during debugging or development:

#include <iostream>

int main() {
    std::cout << "This will print" << std::endl;
    
    // std::cout << "This is commented out" << std::endl;
    
    /*
    std::cout << "This entire block" << std::endl;
    std::cout << "is commented out" << std::endl;
    */
    
    std::cout << "Program continues" << std::endl;
    
    return 0;
}

Common Mistakes

  • Nesting multi-line comments (not allowed in C++)
  • Forgetting to close multi-line comments
  • Using incorrect comment syntax
  • Writing comments that contradict the code

2. What is the correct way to end a multi-line comment?

  • //

  • Yes, they can be nested
  • No, nesting is not allowed
  • Only single-line comments can be nested

  • Explaining complex algorithms
  • Stating the obvious (like "i = i + 1; // add 1 to i")
  • Writing the entire program logic in comments
  • Replacing variable names with comments

5. What happens if you forget to close a multi-line comment?

  • The compiler ignores it
