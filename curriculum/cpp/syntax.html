C++ Syntax

C++ syntax refers to the rules that govern how C++ programs are written. Understanding these rules is essential for writing correct and readable C++ code. C++ has a structured syntax that combines elements from C with object-oriented features.

Basic Structure of a C++ Program

A C++ program consists of several key elements:

  • Preprocessor directives: Lines starting with #
  • Function definitions: Blocks of code that perform tasks
  • Statements: Individual instructions
  • Expressions: Combinations of operators and operands

Statements and Semicolons

In C++, statements are the basic units of execution. Every statement must end with a semicolon (;).

#include <iostream>

int main() {
    std::cout << "Hello";        // Statement 1
    std::cout << " World!";     // Statement 2
    return 0;                     // Statement 3
}

Case Sensitivity

C++ is case-sensitive, meaning uppercase and lowercase letters are treated as different characters.

// These are all different identifiers:
int age;
int Age;
int AGE;

// Keywords must be lowercase:
int main()    // Correct
Int main()    // Error: Int is not a keyword

Whitespace and Indentation

Whitespace (spaces, tabs, newlines) is generally ignored by the compiler, but proper indentation improves readability.

// Both are valid, but the second is more readable:
int main(){std::cout<<"Hello";return 0;}

int main() {
    std::cout << "Hello";
    return 0;
}

Code Blocks

Code blocks are enclosed in curly braces {}. They define scope and group statements together.

int main() {
    // This is a code block
    {
        int x = 5;  // x is only accessible within this block
        std::cout << x << std::endl;
    }
    // x is not accessible here
    return 0;
}

Identifiers

Identifiers are names given to variables, functions, classes, etc. Rules for identifiers:

  • Can contain letters, digits, and underscores
  • Must start with a letter or underscore
  • Cannot be a C++ keyword
  • Are case-sensitive
  • Should be meaningful and descriptive
// Valid identifiers:
int user_age;
double _temperature;
std::string firstName;
char grade_A;

// Invalid identifiers:
// int 123abc;     // Starts with digit
// int my-age;     // Contains hyphen
// int int;        // Reserved keyword

Keywords

Keywords are reserved words that have special meaning in C++. They cannot be used as identifiers.

Common C++ keywords:

  • Data types: int, double, char, bool, void
  • Control flow: if, else, for, while, switch, case
  • Functions: return, main
  • Classes: class, public, private, protected
  • Memory: new, delete
  • Others: const, static, namespace, using

Literals

Literals are constant values written directly in the code:

  • Integer literals: 42, 0x2A (hex), 052 (octal)
  • Floating-point literals: 3.14, 2.5e-3
  • Character literals: 'A', '\n', '\t'
  • String literals: "Hello, World!"
  • Boolean literals: true, false

Expressions

Expressions are combinations of operators, operands, and function calls that evaluate to a value.

// Arithmetic expressions:
int result = 5 + 3 * 2;  // Evaluates to 11

// Logical expressions:
bool is_adult = age >= 18;

// Function call expressions:
std::cout << "Value: " << calculate_sum(5, 3);

Operators

C++ has various operators for different operations:

  • Arithmetic: +, -, *, /, %
  • Assignment: =, +=, -=, *=, /=
  • Comparison: ==, !=, <, >, <=, >=
  • Logical: &&, ||, !
  • Increment/Decrement: ++, --

Preprocessor Directives

Lines starting with # are processed before compilation:

#include <iostream>    // Include header file
#define PI 3.14159     // Define constant
#ifdef DEBUG          // Conditional compilation
    // Debug code
#endif

Namespaces

Namespaces help organize code and prevent name conflicts:

// Without using namespace:
std::cout << "Hello" << std::endl;

// With using namespace:
using namespace std;
cout << "Hello" << endl;

// Using specific items:
using std::cout;
using std::endl;
cout << "Hello" << endl;

Example Program

#include <iostream>
#include <string>

using namespace std;

int main() {
    // Variable declarations
    int age = 25;
    double height = 5.9;
    string name = "Alice";
    
    // Expressions and statements
    int next_year = age + 1;
    bool is_tall = height > 6.0;
    
    // Output statements
    cout << "Name: " << name << endl;
    cout << "Age: " << age << endl;
    cout << "Next year: " << next_year << endl;
    cout << "Is tall: " << (is_tall ? "Yes" : "No") << endl;
    
    return 0;
}

Mini Project

Create a C++ program that demonstrates proper syntax:

  • Use proper indentation and code blocks
  • Include preprocessor directives
  • Declare and use variables with meaningful names
  • Use different types of literals
  • Write clear, readable statements
  • Bonus: Use namespaces appropriately

Common Syntax Errors

  • Missing semicolons: Every statement needs ;
  • Mismatched braces: { must match }
  • Wrong case: main() not Main()
  • Missing includes: Need headers for used functions
  • Invalid identifiers: Cannot start with digit or use keywords

Best Practices

  • Use consistent indentation (4 spaces or 1 tab)
  • Write one statement per line
  • Use meaningful variable names
  • Add comments for complex logic
  • Keep functions reasonably sized
  • Use proper naming conventions (camelCase for variables)
