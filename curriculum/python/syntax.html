Syntax

Python syntax refers to the rules that define how Python programs are written and interpreted. Python's syntax is designed to be clean, readable, and easy to learn, emphasizing code readability over complex syntax rules.

Basic Structure

Python programs are composed of statements, which are executed sequentially. Unlike many other languages, Python uses indentation to define code blocks instead of braces or keywords.

Indentation

Indentation is crucial in Python:

  • Used to define code blocks (functions, loops, conditionals)
  • Typically 4 spaces per level (or a tab)
  • Must be consistent throughout the program
  • Incorrect indentation causes IndentationError
# Correct indentation
if True:
    print("This is indented correctly")
    if False:
        print("Nested indentation")

# Incorrect indentation (would cause error)
# if True:
# print("This will cause an IndentationError")

Statements and Lines

  • Each statement typically occupies one line
  • Statements end at the end of the line (no semicolon required)
  • Long statements can be continued using backslash (\) or parentheses
  • Multiple statements can be on one line using semicolons (not recommended)
# Single line statements
x = 5
print("Hello")

# Line continuation with backslash
total = x + \
        y + \
        z

# Line continuation with parentheses
result = (x +
          y +
          z)

# Multiple statements on one line (not recommended)
x = 5; y = 10; print(x + y)

Comments

Comments are used to explain code and are ignored by the interpreter:

  • Single-line comments start with #
  • Multi-line comments use triple quotes (""" or ''')
  • Comments should be clear and helpful
# This is a single-line comment
print("Hello")  # This comment is inline

"""
This is a multi-line comment
that spans several lines
"""
print("World")

'''
Another way to write
multi-line comments
'''

Case Sensitivity

Python is case-sensitive:

  • Variable names, function names, and keywords are case-sensitive
  • print and Print are different identifiers
  • True and true are different (True is a boolean, true is undefined)

Keywords

Python has reserved keywords that cannot be used as identifiers:

Basic Program Structure

A typical Python program includes:

  • Import statements at the top
  • Function and class definitions
  • Main execution code
  • Proper indentation throughout
# Import statements
import math

# Function definition
def greet(name):
    return f"Hello, {name}!"

# Main execution
if __name__ == "__main__":
    message = greet("World")
    print(message)

Common Syntax Errors

  • IndentationError: Inconsistent indentation
  • SyntaxError: Invalid Python syntax (missing colons, parentheses, etc.)
  • NameError: Using undefined variables
  • TypeError: Operations on incompatible types

Best Practices

  • Use 4 spaces for indentation (not tabs)
  • Keep lines under 79 characters (PEP 8 guideline)
  • Use meaningful variable and function names
  • Add comments to explain complex logic
  • Follow consistent naming conventions

Example Program

# This program demonstrates basic Python syntax
def calculate_area(length, width):
    """
    Calculate the area of a rectangle.
    
    Args:
        length (float): The length of the rectangle
        width (float): The width of the rectangle
        
    Returns:
        float: The area of the rectangle
    """
    area = length * width
    return area

# Main program
if __name__ == "__main__":
    # Get user input
    length = 5.0
    width = 3.0
    
    # Calculate area
    result = calculate_area(length, width)
    
    # Display result
    print(f"The area of the rectangle is: {result}")
