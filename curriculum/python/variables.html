Variables

Variables are containers for storing data values in Python. They allow you to store information that can be referenced and manipulated throughout your program. In Python, variables are created when you assign a value to them.

What are Variables?

  • Named storage locations in memory
  • Used to hold data that can change during program execution
  • Make code more readable and maintainable
  • Can store different types of data (numbers, strings, etc.)
  • Values can be reassigned during program execution

Variable Naming Rules

Python has specific rules for naming variables:

  • Variable names must start with a letter or underscore (_)
  • The rest of the name can contain letters, numbers, and underscores
  • Variable names are case-sensitive (age, Age, and AGE are different)
  • Cannot use Python keywords as variable names
  • No spaces allowed in variable names

Good Variable Naming Practices

  • Use descriptive names that indicate the variable's purpose
  • Use lowercase letters with underscores for multi-word names (snake_case)
  • Avoid single-letter names except for simple counters (i, j, k)
  • Keep names concise but meaningful
  • Be consistent throughout your code

Examples

# Valid variable names
name = "Alice"
age = 25
is_student = True
user_name = "bob123"
_total_score = 95

# Invalid variable names (would cause errors)
# 123name = "invalid"  # Starts with number
# class = "Python"     # Reserved keyword
# user-name = "bob"    # Contains hyphen

# Variable assignment and reassignment
x = 10
print(x)  # Output: 10

x = 20  # Reassigning the value
print(x)  # Output: 20

# Multiple assignment
a = b = c = 100
print(a, b, c)  # Output: 100 100 100

# Assigning multiple values
name, age, city = "Alice", 25, "New York"
print(name)  # Output: Alice
print(age)   # Output: 25
print(city)  # Output: New York

Dynamic Typing

Python uses dynamic typing, which means:

  • You don't need to declare the data type when creating a variable
  • The data type is determined automatically based on the assigned value
  • Variables can change type during execution
  • This makes Python flexible but requires careful type management
# Dynamic typing example
x = 5        # x is an integer
print(type(x))  # Output: 

x = "Hello"  # Now x is a string
print(type(x))  # Output: 

x = 3.14     # Now x is a float
print(type(x))  # Output:

Variable Scope

Variables can have different scopes:

  • Local variables: Defined inside a function, only accessible within that function
  • Global variables: Defined outside functions, accessible throughout the program
  • Instance variables: Defined within a class, unique to each object instance

Constants

Although Python doesn't have built-in constant types, you can indicate constants by:

  • Using all uppercase letters for the variable name
  • Placing them at the top of the file
  • Not modifying them throughout the program
PI = 3.14159
MAX_USERS = 100
CONFIG_FILE = "settings.ini"

Mini Project

Create a simple calculator program that demonstrates variable usage:

  • Define variables for two numbers
  • Perform basic arithmetic operations (addition, subtraction, multiplication, division)
  • Store results in separate variables
  • Print the results with descriptive messages
  • Bonus: Ask the user for input values

Here's a starting template:

# Define variables
num1 = 10
num2 = 5

# Perform calculations
addition = num1 + num2
subtraction = num1 - num2
multiplication = num1 * num2
division = num1 / num2

# Print results
print(f"Addition: {num1} + {num2} = {addition}")
print(f"Subtraction: {num1} - {num2} = {subtraction}")
print(f"Multiplication: {num1} * {num2} = {multiplication}")
print(f"Division: {num1} / {num2} = {division}")

Common Mistakes

  • Forgetting to initialize variables before use
  • Using undefined variables
  • Misspelling variable names (case-sensitive)
  • Using reserved keywords as variable names
  • Not following naming conventions
