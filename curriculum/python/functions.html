Functions

Functions are reusable blocks of code that perform specific tasks. They allow you to organize your code into logical units, avoid repetition, and make your programs more modular and maintainable. In Python, functions are first-class objects that can be passed around, stored in variables, and even defined inside other functions.

Why Use Functions?

Functions provide several benefits:

  • Code Reusability: Write once, use many times
  • Modularity: Break complex problems into smaller, manageable pieces
  • Readability: Make code easier to understand and maintain
  • Testing: Test individual components separately
  • Abstraction: Hide implementation details
  • Collaboration: Different developers can work on different functions

Defining Functions

The basic syntax for defining a function:

def function_name(parameters):
    """Optional docstring describing the function"""
    # Function body
    # Code to execute
    return result  # Optional return statement

Key Components:

  • def: Keyword that defines a function
  • function_name: Identifier for the function (follows variable naming rules)
  • parameters: Optional input parameters
  • docstring: Optional documentation string
  • return: Optional statement to return a value

Simple Function Example

def greet():
    """Print a simple greeting"""
    print("Hello, World!")

# Call the function
greet()  # Output: Hello, World!

Functions with Parameters

Parameters allow functions to accept input values:

def greet_person(name):
    """Greet a specific person"""
    print(f"Hello, {name}!")

def add_numbers(a, b):
    """Add two numbers and return the result"""
    return a + b

# Using the functions
greet_person("Alice")  # Output: Hello, Alice!
result = add_numbers(5, 3)
print(result)  # Output: 8

Default Parameters

Parameters can have default values:

def greet_with_time(name, time="morning"):
    """Greet someone with a time of day"""
    print(f"Good {time}, {name}!")

greet_with_time("Alice")           # Good morning, Alice!
greet_with_time("Bob", "afternoon") # Good afternoon, Bob!

Note: Default parameters must come after non-default parameters.

Keyword Arguments

You can call functions using parameter names:

def create_profile(name, age, city):
    """Create a user profile"""
    return f"Name: {name}, Age: {age}, City: {city}"

# Positional arguments
profile1 = create_profile("Alice", 25, "New York")

# Keyword arguments
profile2 = create_profile(age=30, name="Bob", city="LA")

# Mixed arguments
profile3 = create_profile("Charlie", city="Chicago", age=35)

print(profile1)
print(profile2)
print(profile3)

Return Values

Functions can return values using the return statement:

def calculate_area(length, width):
    """Calculate the area of a rectangle"""
    area = length * width
    return area

def is_even(number):
    """Check if a number is even"""
    return number % 2 == 0

# Using return values
rect_area = calculate_area(5, 3)
print(f"Area: {rect_area}")  # Area: 15

if is_even(4):
    print("4 is even")  # This will print
else:
    print("4 is odd")

Note: If no return statement is provided, the function returns None.

Multiple Return Values

Functions can return multiple values as a tuple:

def get_user_info():
    """Get user information"""
    name = "Alice"
    age = 25
    email = "alice@example.com"
    return name, age, email

# Unpacking multiple return values
user_name, user_age, user_email = get_user_info()
print(f"Name: {user_name}, Age: {user_age}, Email: {user_email}")

# Or receive as a single tuple
info = get_user_info()
print(info)  # ('Alice', 25, 'alice@example.com')

Variable Scope

Variables have different scopes:

global_var = "I'm global"

def demonstrate_scope():
    local_var = "I'm local"
    print(global_var)  # Can access global
    print(local_var)   # Can access local

demonstrate_scope()
# print(local_var)  # Error: local_var not defined outside function

def modify_global():
    global global_var
    global_var = "I've been modified"

modify_global()
print(global_var)  # Output: I've been modified

Lambda Functions

Anonymous functions for simple operations:

# Regular function
def square(x):
    return x ** 2

# Lambda equivalent
square_lambda = lambda x: x ** 2

print(square(5))        # 25
print(square_lambda(5)) # 25

# Lambda with multiple parameters
add = lambda a, b: a + b
print(add(3, 4))  # 7

# Common use: sorting with key
students = [("Alice", 85), ("Bob", 92), ("Charlie", 78)]
students.sort(key=lambda student: student[1], reverse=True)
print(students)  # Sorted by grade descending

Function Documentation

Docstrings provide documentation for functions:

def calculate_grade(score, total_points=100):
    """
    Calculate the percentage grade and letter grade.

    Args:
        score (float): The points earned
        total_points (float): Total possible points (default 100)

    Returns:
        tuple: (percentage, letter_grade)
    """
    percentage = (score / total_points) * 100
    
    if percentage >= 90:
        letter = "A"
    elif percentage >= 80:
        letter = "B"
    elif percentage >= 70:
        letter = "C"
    elif percentage >= 60:
        letter = "D"
    else:
        letter = "F"
    
    return percentage, letter

# Access docstring
print(calculate_grade.__doc__)

# Use the function
percent, grade = calculate_grade(85, 100)
print(f"Grade: {percent:.1f}%, Letter: {grade}")

Recursive Functions

Functions that call themselves:

def factorial(n):
    """Calculate factorial using recursion"""
    if n == 0 or n == 1:
        return 1
    else:
        return n * factorial(n - 1)

def fibonacci(n):
    """Generate nth Fibonacci number"""
    if n <= 1:
        return n
    else:
        return fibonacci(n - 1) + fibonacci(n - 2)

print(factorial(5))   # 120
print(fibonacci(8))   # 21

Warning: Recursion can cause stack overflow for large inputs.

Mini Project

Create a comprehensive calculator application:

  • Implement basic arithmetic functions (add, subtract, multiply, divide)
  • Create functions for advanced operations (power, square root, factorial)
  • Include input validation and error handling
  • Implement a calculator menu system
  • Use functions to organize code and avoid repetition
  • Bonus: Add memory functions and operation history

Here's a starting template:

def add(a, b):
    """Add two numbers"""
    return a + b

def subtract(a, b):
    """Subtract b from a"""
    return a - b

def multiply(a, b):
    """Multiply two numbers"""
    return a * b

def divide(a, b):
    """Divide a by b"""
    if b == 0:
        raise ValueError("Cannot divide by zero")
    return a / b

def power(base, exponent):
    """Calculate base raised to exponent"""
    return base ** exponent

def square_root(number):
    """Calculate square root using Newton's method"""
    if number < 0:
        raise ValueError("Cannot calculate square root of negative number")
    
    guess = number / 2.0
    for _ in range(10):  # 10 iterations for reasonable accuracy
        guess = (guess + number / guess) / 2.0
    return guess

def factorial(n):
    """Calculate factorial of n"""
    if not isinstance(n, int) or n < 0:
        raise ValueError("Factorial requires non-negative integer")
    
    result = 1
    for i in range(1, n + 1):
        result *= i
    return result

def get_number(prompt):
    """Get a valid number from user input"""
    while True:
        try:
            return float(input(prompt))
        except ValueError:
            print("Invalid input. Please enter a number.")

def display_menu():
    """Display calculator menu"""
    print("\n" + "="*30)
    print("      CALCULATOR MENU")
    print("="*30)
    print("1. Addition")
    print("2. Subtraction")
    print("3. Multiplication")
    print("4. Division")
    print("5. Power")
    print("6. Square Root")
    print("7. Factorial")
    print("8. Exit")
    print("="*30)

def main():
    """Main calculator function"""
    print("Welcome to the Python Calculator!")
    
    while True:
        display_menu()
        
        try:
            choice = input("Enter your choice (1-8): ").strip()
            
            if choice == "8":
                print("Thank you for using the calculator!")
                break
            
            elif choice in ["1", "2", "3", "4", "5"]:
                # Two-operand operations
                operations = {
                    "1": ("Addition", add),
                    "2": ("Subtraction", subtract),
                    "3": ("Multiplication", multiply),
                    "4": ("Division", divide),
                    "5": ("Power", power)
                }
                
                op_name, op_func = operations[choice]
                print(f"\n{op_name}:")
                a = get_number("Enter first number: ")
                b = get_number("Enter second number: ")
                
                result = op_func(a, b)
                print(f"Result: {result}")
            
            elif choice == "6":
                # Square root
                print("\nSquare Root:")
                num = get_number("Enter a non-negative number: ")
                result = square_root(num)
                print(f"Square root of {num} is approximately {result:.6f}")
            
            elif choice == "7":
                # Factorial
                print("\nFactorial:")
                num = int(get_number("Enter a non-negative integer: "))
                result = factorial(num)
                print(f"{num}! = {result}")
            
            else:
                print("Invalid choice. Please select 1-8.")
        
        except ValueError as e:
            print(f"Error: {e}")
        except Exception as e:
            print(f"An unexpected error occurred: {e}")
        
        input("\nPress Enter to continue...")

if __name__ == "__main__":
    main()

Common Mistakes

  • Forgetting to call the function (missing parentheses)
  • Not returning values when needed
  • Using global variables unnecessarily
  • Wrong parameter order when calling functions
  • Mixing positional and keyword arguments incorrectly
  • Infinite recursion without proper base case

Best Practices

  • Use descriptive function names
  • Write docstrings for all functions
  • Keep functions focused on single responsibilities
  • Use default parameters appropriately
  • Avoid global variables when possible
  • Test functions independently
  • Use type hints for better code clarity (Python 3.5+)
